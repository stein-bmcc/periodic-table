    meta

    The meta element represents various kinds of metadata that cannot be expressed using the title, base, link, style, and script elements.

    The meta element can represent document-level metadata with the name attribute, pragma directives with the http-equiv attribute, and the file's character encoding declaration when an HTML document is serialized to string form (e.g. for transmission over the network or for disk storage) with the charset attribute.

    Meta tags are useful in SEO for helping search engine spiders index your page properly. See more from Google here.

    Example:

    This shows a number of examples of different attributes for meta tags.

    • charset: this defines the character set used. Important for languages other than English. UTF-8 works for most languages so is a good default value.
    • http-equiv: this is for Internet Explorer (IE) and allows you to determine which mode IE will render in. Edge tells IE to render as normal. You can see a longer explanation and MS explanation
    • name: the name="description" is used by search engines like Google to read a short description of the page. see more from Google
    • name: the name="viewport" tag is a key part of responsive design and helps ensure mobile browsers will render the page as you expect (and not shrink it down). This is important when doing Responsive Web Design

    link

    The link element allows authors to link their document to other resources.

    The destination of the link(s) is given by the href attribute, which must be present and must contain a valid non-empty URL potentially surrounded by spaces. If the href attribute is absent, then the element does not define a link.

    A link element must have rel attribute.

    div

    A div is a generic element used to group elements together. Use it when one of the other grouping elements (article, section, aside, nav etc) are not appropriate.

    Because the div has no meaning by itself, you should always (almost) use it with one or more classes. Otherwise it is hard to use CSS to select the div and hard to know what it is grouping.

    One of class names should describe the content in the div. You can kind of think of div as a way to add your own custom elements to the page.

    Example:

    In this case the developer needed to create a page with a number of speakers. Two divs are used to mark a group of elements as belonging to a speaker (div class=speaker) and the image of the speaker (div class=pic).

    pop-uptext
    1.   <div class="speaker">
    2.       <div class="pic">
    3.          <img src="http://placehold.it/100x100" alt="picture of speaker">
    4.       </div> <!-- end .pic -->
    5.      <h3>Name of Speaker</h3>
    6.      <p>Bio of Speaker</p>
    7.   </div><!-- end .speaker -->
     <div class="speaker">
     <div class="pic">
         <img src="http://placehold.it/100x100" alt="picture of speaker">
     </div> <!-- end .pic -->
     <h3>Name of Speaker</h3>

 </div><!-- end .speaker -->

    h1, h2, h3, h4, h5, h6

    Represent headings and subheadings. These elements rank in importance according to the number in their name. The h1 element is said to have the highest rank, the h6 element has the lowest rank, and two elements with the same name have equal rank.

    span

    The HTML <span> element is a generic inline container for phrasing content, which does not inherently represent anything. It can be used to group elements for styling purposes (using the class or id attributes), or because they share attribute values, such as lang. It should be used only when no other semantic element is appropriate. <span> is very much like a <div> element, but <div> is a block-level element whereas a <span> is an inline element.

    source MDN

    <p>Gradually add the <span class="ingredient">olive oil</span> 
while running the blender slowly.</p>

    b

    The HTML Bring Attention To element (<b>) is used to draw the reader's attention to the element's contents, which are not otherwise granted special importance. This was formerly known as the Boldface element, and most browsers still draw the text in boldface. However, you should not use <b> for styling text; instead, you should use the CSS font-weight property to create boldface text, or the <strong> element to indicate that text is of special importance.

    source MDN

    <p>
  This article describes several <b class="keywords">text-level</b> elements.
</p>

    li

    The HTML <li> element is used to represent an item in a list. It must be contained in a parent element: an ordered list (<ol>), an unordered list (<ul>), or a menu (<menu>). In menus and unordered lists, list items are usually displayed using bullet points. In ordered lists, they are usually displayed with an ascending counter on the left, such as a number or letter.

    source MDN

  <li>left foot slide</li>
  <li>Left foot up,</li>
  <li>right foot slide</li>
</ol>

    The HTML <dl> element represents a description list. The element encloses a list of groups of terms (specified using the <dt> element) and descriptions (provided by <dd> elements). Common uses for this element are to implement a glossary or to display metadata (a list of key-value pairs).

    img

    The HTML <img> element embeds an image into the document.The src attribute is required, and contains the path to the image you want to embed. The alt attribute holds a text description of the image, which isn't mandatory but is incredibly useful for accessibility. Screen readers read this description out to their users so they know what the image means.

    source MDN

    <img src="images/mydoc.jpg" alt="dog with a bone">
